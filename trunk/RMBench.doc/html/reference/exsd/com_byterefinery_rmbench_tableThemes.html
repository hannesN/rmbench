Table Themes

Identifier:
com.byterefinery.rmbench.tableThemes

Since:
1.0

Description:
This extension point allows the definition of table color schemes (aka themes). A theme can redefine the background and foreground colors for the title and the body area of a table figure. It can then be associated with a table type to cause the thus typed tables to be shown with the given color scheme.

Configuration Markup:

<!ELEMENT extension (theme+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT theme EMPTY>

<!ATTLIST theme

titleBackground CDATA #IMPLIED

titleForeground CDATA #IMPLIED

bodyBackground  CDATA #IMPLIED

bodyForeground  CDATA #IMPLIED

id              CDATA #REQUIRED

label           CDATA #IMPLIED

>

a theme represents a group of color definitions that can be associated with a table type and thus be applied to a table


  • titleBackground - the title background color as a comma-separated RGB value
  • titleForeground - the title foreground color as a comma-separated RGB value
  • bodyBackground - the table body background color as a comma-separated RGB value
  • bodyForeground - the table body foreground color as a comma-separated RGB value
  • id - a unique ID for this element
  • label - a displayable, translatable label. If absent, the ID will be used for presentation purposes

Examples: 

   <extension
         point="com.byterefinery.rmbench.tableThemes">
      <theme
            bodyBackground="229,229,229"
            id="theme1"
            label="Theme 1"
            titleBackground="50,195,50"/>
      <theme
            bodyBackground="255,255,0"
            id="theme2"
            label="Theme 2"
            titleBackground="255,165,0"/>
   </extension>

Supplied Implementation:
RMBench comes with a set of table themes and table types which can be used to distinguish tables in a model
