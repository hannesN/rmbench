Native Database info

Identifier:
com.byterefinery.rmbench.databaseInfo

Since:
1.0

Description:
This extension point is used to add support for specific database products

Configuration Markup:

<!ELEMENT extension (database+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT database (extraTypeEditor*)>

<!ATTLIST database

id              CDATA #REQUIRED

name            CDATA #REQUIRED

className       CDATA #REQUIRED

messageProvider CDATA "com.byterefinery.rmbench.external.database.DBMessageProvider"

>

description of a native database


  • id - a unique identifier for this element
  • name - the displayable and translatable name for this element
  • className - the name of a class that implements com.byterefinery.rmbench.extension.IDatabaseInfo interface
  • messageProvider - name of a class that implements the com.byterefinery.rmbench.external.IMessageProvider interface. By default, this will be set to com.byterefinery.rmbench.external.database.DBMessageProvider

<!ELEMENT extraTypeEditor EMPTY>

<!ATTLIST extraTypeEditor

type    CDATA #REQUIRED

factory CDATA #REQUIRED

>

an association between a data type identifier and a custom type editor which is used to configure the type in a database-specific manner


  • type - the identifier of a data type registered with this database
  • factory - name of a class that implements the com.byterefinery.rmbench.external.IDataTypeEditorFactory interface. Only one instance of this class will be created. It is responsible for creating editors as required

Examples: 

<extension point="com.byterefinery.rmbench.databaseInfo">
 <database className="com.mycorp.rmbench.SuperDatabase" name="SuperDB"/>
</extension>

API Information:
The class named in the className attribute must implement the com.byterefinery.rmbench.external.IDatabaseInfo interface.

The class named in the optional messageProvider attribute must implement the com.byterefinery.rmbench.external.IMessageProvider interface.

Supplied Implementation:
The RMBench base product comes with implementations for a number of common database products, such as Oracle and PostgreSQL. There is also a generic implementation that mirrors the SQL:99 standard.
