Table Types

Identifier:
com.byterefinery.rmbench.tableTypes

Since:
1.0

Description:
table types provide a means to assign symbolic names to groups of tables, for use in presentation and code generation.

Configuration Markup:

<!ELEMENT extension (type+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT type EMPTY>

<!ATTLIST type

id    CDATA #REQUIRED

label CDATA #IMPLIED

theme CDATA #IMPLIED

>

a table type is a symbolic name that can be assigned to a group of tables, for use in presentation and code generation.


  • id - a unique ID for this element
  • label - a displayable, translatable label. If absent, the ID will be used for presentation purposes
  • theme - the name of a theme that is associated with this type. Tables that bear this type are shown are shown in diagrams as specified by the theme.

Examples: 

   <extension
         point="com.byterefinery.rmbench.tableTypes">
      <type
            id="lookup"
            label="Lookup"
            theme="theme1"/>
      <type
            id="associative"
            label="Associative"
            theme="theme2"/>
   </extension>

Supplied Implementation:
RMBench provides the following predefined table types:
  • Lookup. A table with lookup values (e.g., currency, country) that are seldom changed
  • Associative. An table that associates 2 or more entities in the physical model
