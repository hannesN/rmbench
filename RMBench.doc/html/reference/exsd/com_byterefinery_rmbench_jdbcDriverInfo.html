Jdbc Driver Info

Identifier:
com.byterefinery.rmbench.jdbcDriverInfo

Since:
1.0

Description:
this extension point allows the integration of JDBC drivers for use during the reverse and forward engineering processes. A JDBC driver is associated with a database product and must provide data about its required configuration

Configuration Markup:

<!ELEMENT extension (driver+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT driver ((setupWidget | genericURL | jdbcURL) , databaseMetaData? , connectAdapter?)>

<!ATTLIST driver

id            CDATA #REQUIRED

name          CDATA #REQUIRED

className     CDATA #REQUIRED

database      CDATA #REQUIRED

queryUserId   (true | false) "true"

queryPassword (true | false) "true"

>

description of a JDBC driver for a specific database product


  • id - a unique ID for this element
  • name - the user-visible name of this driver
  • className - the name of the class that implements the JDBC driver
  • database - the ID of a database extension which describes the native database this driver connects to. The name must resolve to a com.byterefinery.rmbench.databaseInfo extension declared elsewhere
  • queryUserId - a flag whether a login id is needed for the connection or not
  • queryPassword - a flag whether a password is needed for the connection or not

<!ELEMENT setupWidget EMPTY>

<!ATTLIST setupWidget

factory CDATA #REQUIRED

>

this element allows full control over the creation of widgets used in the wizard dialog during the setup of a database connection. Either this element or a xxxURL element must be present


  • factory - class name of a factory class that is used to create the setup control. Must extend the com.byterefinery.rmbench.extension.URLSetupGroup.Factory class

<!ELEMENT genericURL (variable*)>

<!ATTLIST genericURL

pattern CDATA #IMPLIED

>

this element allows to describe the connection URL with all variables expected by the driver. Appropriate widgets will be created dynamically at runtime to configure the database connection


  • pattern - a pattern describing the URL. The pattern can contain variable references designated by the ${name} syntax. Each variable that appears in the URL pattern must be further described in a variable subelement.

<!ELEMENT variable EMPTY>

<!ATTLIST variable

name         CDATA #REQUIRED

resourceName CDATA #IMPLIED

type         (int|java-name|string|boolean)

use          (optional|required)

default      CDATA #IMPLIED

>

Description of a variable that appears in the URL pattern


  • name - the name of the variable - exactly as it appears between the ${} delimiters in the pattern
  • resourceName - the name under which the field label is looked up from the resource. If this attribute is used, the name given should correspond to an entry in the resource bundle of the declaring plugin. The value resulting from the lookup will be used as field label at runtime. If absent, the the variable name will be used as label.
  • type - the data type of the variable, used for field validation. One of int, java-name, string or boolean
  • use - determines whether this is an optional or required variable
  • default - a default value for the variable. Must be compatible with the type

<!ELEMENT databaseMetaData EMPTY>

<!ATTLIST databaseMetaData

factory CDATA #IMPLIED

>

allows specifying a database metadata adapter for this driver


  • factory - name of the factory class for creating the database metadata adapter. It must implement the interface com.byterefinery.rmbench.extension.IMetaDataAccess.Factory

<!ELEMENT jdbcURL EMPTY>

<!ATTLIST jdbcURL

defaulthost CDATA #IMPLIED

protocol    CDATA #IMPLIED

defaultport CDATA #IMPLIED

defaultdb   CDATA #IMPLIED

>

This element describes a standard jdbc url in the form: jdbc:protocol:[//host[:port]/]database.


  • defaulthost - The default host used by the jdbc url.
  • protocol - The protocol used by the url, usually it't the name of the database, e.g. mysql its the string between jdbc: and the following : This would result in an url like jdbc:mysql: ...
  • defaultport - The default port used by the jdbc url.
  • defaultdb - The default database used by the jdbc url.

<!ELEMENT connectAdapter EMPTY>

<!ATTLIST connectAdapter

factory CDATA #IMPLIED

>

allows specification of a connect adapter


  • factory - this attribute specifies a factory for connect adapters. It must implement the com.byterefinery.rmbench.external.database.jdbc.IJdbcConnectAdapter.Factory interface. A connect adapter is used to intercept connect and close invocations.

Examples: 
<extension point="com.byterefinery.rmbench.jdbcDriverInfo">
 <driver className="org.superdb.Driver" database="SuperDB" name="SuperDBJdbc">
  <setupURL pattern="jdbc:superdb://${host}:${port}/${database}">
   <variable name="host" 
    default="localhost" 
    resourceName="jdbc_host"
    type="string"
    use="required"/>
   <variable
    name="port"
    default="5432"
    resourceName="jdbc_port"
    type="int"
    use="optional"/>
   <variable
    name="database"
    resourceName="jdbc_database"
    type="string"
    use="required"/>
  </setupURL>
 </driver>
</extension>

Supplied Implementation:
RMBench comes with driver configurations for different database products, like Oracle and PostgreSQL
