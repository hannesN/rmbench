Working with a Diagram

Creating the Diagram

A Diagram is created by using the context menu in Model View. You will see the diagram creation wizard:


In this wizard, you need to specify the name of the new diagram, and the target schema for database elements that are created in the context of the diagram (you can always change the schema for the individual elements later). You can either choose an existing schema, or create one on-the-fly.

After you have created the new diagram, you will see the diagram editor automatically open. On the left, it shows the palette which contains templates of all the database elements that can be created. The image below shows the editor in maximized mode.


Creating a Table

To create a new table, click the left mouse button on the table template (), place the mouse over the desired location in the diagram and click again. An empty table will appear, with a default table name already generated.

The details of the table can be inspected and modified in the Table Details View. If you are working in the default perspective configuration, the details view will already be open in the bottom right part of your workbench page. If it is not open, you can open it by double-clicking on the new table.

Creating a Foreign Key

To create a foreign key, click on the foreign key template () in the palette. As soon as you move the cursor over the diagram, the cursor will change to a connection symbol. Click on the table you want to designate as the source of the relationship, and subsequently on the target table (the target table must hold a primary key). When you release the button, the relationship is created, adding the columns that mirror the target key to the source table.

There are other ways to create a foreign key based on existing columns. Please refer to the tasks section for details.

Creating an Index

To create an index, click the index template () in the palette and place the mouse above a table for which you want to create the index and which contains at least one column. As soon as you click the mouse, the index editor dialog will be opened to let you specify the relevant information for finishing the index creation.

See the tasks section for additional details on creating indices.

Creating a Constraint

You can add a constraint in the same manner by clicking on the constraint symbol () and subsequently on the table to which the constraint will apply. The constraint wizard will let you choose the type of constraint, and the constraint details. Note that foreign key constraints are handled separately, s described above, and therefore do not appear in this dialog.

See the tasks section for additional details on creating constraints.
